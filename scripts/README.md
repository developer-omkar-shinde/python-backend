# ECS Scripts - Quick Reference

## Files

```
scripts/
├── shutdown_ecs.sh      ← Shutdown all services, tasks, and optionally the ALB
├── startup_ecs.sh       ← Start services and recreate ALB if previously deleted
├── ecs_shutdown.py      ← Advanced shutdown with cost estimation (Python)
├── alb-config.json      ← Auto-generated by --delete-alb (gitignored)
└── README.md            ← This file
```

---

## Quick Start

### Install Dependencies
```bash
# AWS CLI
brew install awscli           # macOS
sudo apt install awscli       # Linux

aws configure
```

### Shutdown (with full cost savings)
```bash
./scripts/shutdown_ecs.sh --delete-alb
```

### Startup
```bash
./scripts/startup_ecs.sh
```

---

## Commands

### Shutdown

| Command | Purpose |
|---------|---------|
| `./scripts/shutdown_ecs.sh` | Scale ECS to 0, deregister ALB targets (ALB keeps billing) |
| `./scripts/shutdown_ecs.sh --delete-alb` | Scale ECS to 0 + delete ALB, save config for restore |
| `./scripts/shutdown_ecs.sh --cluster my-cluster` | Target a specific cluster |
| `./scripts/shutdown_ecs.sh --region us-east-1` | Use a specific region |
| `./scripts/shutdown_ecs.sh --verbose` | Show detailed output |

### Startup

| Command | Purpose |
|---------|---------|
| `./scripts/startup_ecs.sh` | Recreate ALB if deleted, then start ECS service |
| `./scripts/startup_ecs.sh --desired-count 2` | Start 2 task instances |
| `./scripts/startup_ecs.sh --service my-service` | Target a specific service |

### Python Script (Advanced)

| Command | Purpose |
|---------|---------|
| `python3 scripts/ecs_shutdown.py` | Shutdown with detailed logging |
| `python3 scripts/ecs_shutdown.py --estimate-savings` | Show cost estimate without shutting down |
| `python3 scripts/ecs_shutdown.py --verbose` | Enable debug logging |

### Manual AWS CLI

```bash
# List services
aws ecs list-services --cluster python-backend --region us-east-1

# Scale a service up or down
aws ecs update-service --cluster python-backend \
  --service onboarding-service --desired-count 0 --region us-east-1

# Check service status
aws ecs describe-services --cluster python-backend \
  --services onboarding-service --region us-east-1

# View logs
aws logs tail /ecs/backend-learning --follow --region us-east-1

# Get load balancer DNS
aws elbv2 describe-load-balancers --region us-east-1 \
  --query 'LoadBalancers[].{Name:LoadBalancerName,URL:DNSName}'
```

## How ALB Cost Saving Works

The ALB charges ~$0.008/hour (~$5.76/month) just for existing, even with zero traffic.
ECS target groups are free when empty.

The `--delete-alb` flag handles this by:

1. **Shutdown** — saves full ALB config (subnets, security groups, listeners, target group ARNs) to `scripts/alb-config.json`, then deletes the ALB. Target groups are kept so the ECS service definition stays intact.
2. **Startup** — detects `alb-config.json`, recreates the ALB with the exact same config, waits for it to become active, then starts ECS tasks.

> **Note:** When the ALB is recreated its DNS name changes. If you use a custom domain via Route53, update the alias record after startup.

## Cost Impact

### Always-on (24/7, no shutdown)
```
ECS Fargate:        ~$4.89/month
ALB:               ~$16.20/month
CloudWatch:         ~$2.50/month
─────────────────────────────────
Total:             ~$23.59/month
```

### With shutdown_ecs.sh (no --delete-alb)
```
ECS Fargate:          $0/month   ← tasks stopped
ALB:               ~$5.76/month  ← still billing per hour
CloudWatch:         ~$0.50/month
─────────────────────────────────
Total:              ~$6.26/month
```

### With shutdown_ecs.sh --delete-alb
```
ECS Fargate:          $0/month
ALB:                  $0/month   ← deleted
Target Groups:        $0/month   ← free when empty
CloudWatch:         ~$0.50/month
─────────────────────────────────
Total:              ~$0.50/month
```

## Automation (Cron)

```bash
crontab -e

# Shutdown at 6 PM with full ALB deletion
0 18 * * * /path/to/scripts/shutdown_ecs.sh --delete-alb >> /tmp/ecs_shutdown.log 2>&1

# Startup at 9 AM (recreates ALB automatically if config exists)
0 9 * * * /path/to/scripts/startup_ecs.sh >> /tmp/ecs_startup.log 2>&1
```

## Troubleshooting

### "command not found: aws"
```bash
brew install awscli
```

### "AWS credentials not configured"
```bash
aws configure
```

### "Cluster not found"
```bash
aws ecs list-clusters --region us-east-1
```

### ALB DNS changed after startup
If you use Route53 or a custom domain, update the ALB alias record after startup:
```bash
# Get new DNS name
aws elbv2 describe-load-balancers --region us-east-1 \
  --query 'LoadBalancers[?contains(LoadBalancerName, `backend-learning`)].DNSName'
```

### Scripts don't seem to do anything
```bash
./scripts/shutdown_ecs.sh --verbose
aws ecs list-services --cluster python-backend --region us-east-1
```

---

## Safety Notes

- Scripts are safe to run multiple times
- Shutdown does **not** delete task definitions, target groups, or databases
- ALB config is saved before deletion — startup will restore it
- `scripts/alb-config.json` is gitignored (contains infrastructure ARNs)
